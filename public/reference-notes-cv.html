01 What is Computer Vision?

Computer vision is a field of artificial intelligence that enables machines to interpret and understand visual information from the world — such as images and video. Rather than humans manually analysing images, models learn patterns from large datasets to make predictions automatically.

There are two core tasks you must understand:

Task 1
Classification
Assigns a label to the entire image. Example: "This image is glaucoma" or "This image is normal."
Task 2
Detection
Locates where something is in the image using a bounding box, then classifies it. More specific than classification.
Key distinction

Classification answers: "What is in this image?"
Detection answers: "What is in this image, and where is it?"
For medical imaging tasks like glaucoma screening, classification is often sufficient — you need to know if glaucoma is present, not pinpoint a bounding box.

02 The Detection Pipeline

Every detection system follows the same fundamental sequence. You must be able to describe each stage and explain what happens between them.

  1. 1
    Input image
    A raw image file (e.g. a fundus photograph of the eye) is loaded into the system. It may be in JPEG, PNG, or DICOM format.
  2. 2
    Preprocessing
    The image is prepared for the model. This includes resizing to a fixed dimension (e.g. 224×224 px), normalising pixel values (typically to 0–1), and sometimes applying augmentations like flipping or contrast adjustment.
  3. 3
    Feature extraction
    The model learns to detect patterns — edges, textures, shapes, and spatial relationships — from labelled training data. In deep learning, convolutional layers do this automatically.
  4. 4
    Model inference
    The trained model processes the image and produces a raw output (logits or probabilities). This is the prediction step — the model is not learning here, only applying what it learned.
  5. 5
    Output / prediction
    The final result: a class label (glaucoma / normal), a confidence score (e.g. 0.87), and optionally a bounding box or heatmap showing where the model focused.
03 What Models Learn From Images

Models do not "see" images the way humans do. They process pixel values and learn to associate patterns with labels during training. The features below are what a well-trained model learns to detect:

EdgesSharp transitions between pixel intensitiesBoundary of the optic disc
TexturesRepeated local patterns across a regionNerve fibre layer texture
ShapesGeometric structures in the imageCup-to-disc ratio (CDR)

Clinical context

In glaucoma detection, the most important feature is the cup-to-disc ratio (CDR). A CDR above 0.6 is typically considered suspicious. The model learns to estimate this ratio from the image automatically.

04 Understanding Model Outputs

When you run a model, it returns structured outputs. You need to know how to read and interpret these correctly — this is tested in Level 2 and Level 3.

Output type 1
Class label
A categorical prediction: Glaucoma or Normal. Derived by applying a threshold (usually 0.5) to the confidence score.
Output type 2
Confidence score
A probability between 0 and 1. Example: 0.87 means the model is 87% confident the image shows glaucoma.
Output type 3
Bounding box
Co-ordinates (x, y, width, height) defining where the model found the relevant region. Used in object detection tasks.
Output type 4
Heatmap / CAM
A visual overlay showing which regions most influenced the prediction. Useful for interpretability and clinical trust.
Important — threshold effect

If a model outputs 0.51, it predicts glaucoma (above 0.5 threshold). If it outputs 0.49, it predicts normal. These two predictions are very close in confidence but result in different labels. In medical contexts, clinicians often lower the threshold (e.g. to 0.3) to catch more true positives — even at the cost of more false positives.

Critical for glaucoma detection

Prioritise Recall (Sensitivity) above all other metrics. In medical screening, a false negative — telling a patient they are normal when they have glaucoma — causes irreversible harm. A false positive only leads to an unnecessary follow-up. The cost of missing a case is always higher than the cost of over-referring.

06 Level 2 — Applying the Workflow
Level 2 · Apply

In Level 2, you will follow a guided 5-step workflow. Use this section to understand what each step requires and why it matters — so you can reason about your choices, not just click through.

  1. 1
    Dataset familiarisation
    Look at sample fundus images. Before running any model, you must visually understand the data. Ask: Are images consistent in quality? Any obvious differences between glaucoma and normal cases? What do the optic discs look like?
  2. 2
    Preprocessing
    Load the image, resize to the required input shape (e.g. 224×224), and normalise pixel values. Without proper preprocessing, the model may produce unreliable outputs. Normalisation (dividing by 255) ensures consistent input scale.
  3. 3
    Run the model
    Pass the preprocessed image to the pre-built model. Record the raw output: class label and confidence score. Do not modify the model — your task is to use it correctly and interpret its output.
  4. 4
    Interpret the prediction
    This is the most important step. Given the prediction: Is the confidence score high or low? Does the prediction seem clinically reasonable? If the model says 0.53 (glaucoma), that is a low-confidence prediction — flag it as uncertain. Explain your reasoning.
  5. 5
    Basic evaluation
    Given a small set of results with ground truth labels, calculate accuracy manually. Identify which cases were incorrect and suggest why (e.g. low image quality, borderline CDR, unusual presentation).
Examiner tip

Level 2 rewards reasoning. You will not be penalised for a wrong prediction from the model. You will be evaluated on whether you correctly interpret what the output means and whether your explanation is clinically logical.

07 Level 3 — Independent Challenge
Level 3 · Prove
Problem statement

You are tasked with developing a detection workflow to identify signs of glaucoma from retinal fundus images. You will be given a dataset of labelled images. Your objective is to independently apply a detection pipeline to classify each image as glaucoma or normal. No step-by-step guidance will be provided.

Use this checklist to ensure your submission is complete:

  1. ✓
    Data handling
    Load the dataset. Inspect image count, class distribution (are there more normal than glaucoma cases?), and image quality. Document any preprocessing decisions and justify your choices.
  2. ✓
    Execution
    Run your complete detection pipeline independently. Ensure your preprocessing matches what the model expects. Record all outputs — do not cherry-pick results.
  3. ✓
    Evaluation
    Calculate accuracy, precision, recall, F1, and AUC if possible. Present a confusion matrix. Identify which class performed worse and explain why.
  4. ✓
    Interpretation
    Write a clear, structured interpretation. Summarise model performance, explain key errors, and state what you would do to improve results. Reference clinical context — why does recall matter more than precision here?

08 Level 1 Quiz Preparation
Level 1 · Understand

Practise with these representative questions. Answers are revealed — cover them as you work through each question, then check your reasoning.

Q1 — Concept: What is the primary difference between image classification and object detection?
  • Classification requires more data than detection
  • Classification assigns a label to the whole image; detection also localises the object with a bounding box
  • Detection is only used for medical images
  • They are interchangeable terms
Answer: B — this distinction is foundational. Always link detection to spatial localisation.
Q2 — Concept: Which stage of the pipeline converts a raw image into a format the model can process?
  • Feature extraction
  • Model inference
  • Preprocessing
  • Output interpretation
Answer: C — preprocessing (resizing, normalisation) happens before the model sees the image.
Q3 — Scenario: A glaucoma detection model achieves 95% accuracy on a dataset where 95% of images are labelled "normal". Should you trust this result? Why or why not?
  • Yes — 95% accuracy is excellent by any standard
  • No — a model that always predicts "normal" would also achieve 95% accuracy on this imbalanced dataset
  • No — accuracy above 90% is always suspicious
  • Yes — as long as the dataset is large enough
Answer: B — this is a classic class imbalance trap. Always check recall and F1 alongside accuracy.
Q4 — Scenario: Your model predicts glaucoma with a confidence of 0.52. A colleague says this is a reliable positive result. Do you agree?
  • Yes — it crossed the 0.5 threshold so it counts as positive
  • No — 0.52 is a very low-confidence prediction; this case should be flagged as uncertain and referred for clinical review
  • No — any score below 0.7 should be ignored
  • Yes — the model always performs at stated accuracy
Answer: B — threshold crossing does not imply reliability. Interpret confidence scores in context.
Q5 — Reasoning: In a glaucoma screening programme, which metric should you optimise for — precision or recall? Justify your answer.
  • Precision — to avoid unnecessary referrals
  • Recall — because missing a glaucoma case (false negative) can cause irreversible vision loss, which is a worse outcome than an unnecessary follow-up
  • Accuracy — as it captures both aspects equally
  • F1 — as it balances both metrics
Answer: B — this is a clinical reasoning question. The cost asymmetry (FN >> FP in screening) always favours recall.
Q6 — Reasoning: Why is normalising pixel values an important preprocessing step?
  • It makes images look clearer to human reviewers
  • It reduces file size for storage efficiency
  • It ensures all input values are on the same scale, preventing features with large pixel values from dominating model learning
  • It converts colour images to greyscale automatically
Answer: C — normalisation is about stable, consistent input for the model. Link it to the model's learning process.
09 Key Terms Glossary
AUC-ROC
Area Under the Receiver Operating Characteristic Curve. Measures a model's ability to distinguish between classes across all thresholds.
Bounding box
A rectangle defined by (x, y, width, height) that marks the location of a detected object in an image.
Confidence score
A probability (0–1) indicating how certain the model is about its prediction.
Confusion matrix
A table showing TP, TN, FP, FN counts — the foundation for all classification metrics.
Cup-to-disc ratio
Ratio of optic cup diameter to optic disc diameter. A CDR above 0.6 is a clinical indicator of glaucoma risk.
False negative (FN)
Model predicts negative (normal) but the actual label is positive (glaucoma). The most dangerous error in screening.
False positive (FP)
Model predicts positive (glaucoma) but actual label is negative (normal). Leads to unnecessary follow-up.
Feature extraction
The process by which a model learns to identify meaningful patterns (edges, shapes, textures) from training data.
Fundus image
A photograph of the interior surface of the eye (retina). The standard image type used for glaucoma screening.
Inference
Running a trained model on new data to generate predictions. The model is not learning — it is applying learned weights.
Normalisation
Scaling pixel values from their original range (0–255) to a smaller range (0–1) for stable model input.
Precision
Of all cases predicted positive, how many were actually positive. TP / (TP + FP).
Recall (Sensitivity)
Of all actual positive cases, how many were correctly identified. TP / (TP + FN). Critical in screening.
Threshold
The confidence cutoff used to convert a probability into a class label. Default is 0.5; may be lowered to increase recall.
